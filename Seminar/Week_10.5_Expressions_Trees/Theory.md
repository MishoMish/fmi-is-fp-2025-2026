# Двоични дървета и Абстрактни синтактични дървета (AST)

## Въведение

Тази седмица ще разгледаме по-сложни операции над двоични дървета (Двоични наредени дървета - BST) и ще видим едно от най-популярните им приложения – представянето и оценяването на математически изрази чрез абстрактни синтактични дървета (АСТ / AST).

## 1. Абстрактни синтактични дървета (AST) за аритметични изрази

Формалните граматики ни позволяват да дефинираме езици (като езика на математическите изрази).
Например, един прост израз може да се състои от:

- Константи (числа)
- Операции (`+`, `-`, `*`, `/`)
- Променливи (`x`, `y` и т.н.)

### Лексически анализ и Синтактично дърво

Преди да оценим един израз (напр. `(1 + x) * 3`), той се разбива на "токени" (лексически анализ) и после се парсва до синтактично дърво:

```haskell
data Token = Number Int
           | Operator Char
           | OpenParen
           | CloseParen
           | Variable String

data Expression = Constant Int
                | Var String
                | Op Expression Char Expression
```

### Оценка на дървото

Оценката на дървото се състои в неговото обхождане:

```haskell
type Environment = [(String, Int)]

eval :: Environment -> Expression -> Int
-- Базови случаи: дървото е Константа или Променлива
-- Рекурсивна стъпка: дървото е Операция (Op left op right)
```

## 2. Двоични наредени дървета (Binary Search Trees - BST)

При стандартните двоични дървета нямаме условие за наредба. При **двоичните наредени дървета (BST)** важи следното правило за всеки възел `x`:

- Всички елементи в **лявото** поддърво са по-малки от `x`.
- Всички елементи в **дясното** поддърво са по-големи от `x` (при допускане, че няма повтарящи се елементи, или $\ge$ ако има).

Това свойство позволява много по-бързо търсене ($O(\log n)$ средно време).

### Основни операции в BST:

1. **Търсене:** Сравняваме с корена. Ако търсеният елемент е по-малък, търсим в лявото поддърво; ако е по-голям – в дясното.
2. **Вмъкване:** Подобно на търсенето, слизаме надолу докато стигнем пусто дърво (`Empty`) и там поставяме новия елемент.
3. **Изтриване (най-сложната операция):**
   - Ако възелът няма наследници: просто го премахваме.
   - Ако възелът има 1 наследник: заменяме го с наследника му.
   - Ако възелът има 2 наследника: намираме **най-малкия елемент в дясното поддърво**, слагаме неговата стойност на мястото на изтривания възел и рекурсивно изтриваме намерената стойност от дясното поддърво.

```haskell
data IntTree = Empty | Node Int IntTree IntTree
```

## 3. Други основни термини и операции при дървета

- **Листа (Leaves)**: Възлите, които нямат наследници.
- **Ниво (Level)**: Можем да приемем, че коренът е на ниво 0 (или 1). Всички негови директни наследници са на ниво 1, техните наследници на ниво 2 и т.н.
- **Братя (Siblings)**: Възли, които имат един и същи родител.
- **Път от корен до листо (Root-to-leaf path)**: Последователността от възли, тръгвайки от корена надолу до някое от листата.
- **Дървесни ротации (Tree Rotations)**: Операции, при които се променя структурата на дървото, но се запазва наредбата (напр. LL или RR ротация), често използвани при балансиращи се дървета.
- **Подкастряне (Pruning)**: Премахване на всички възли след определено ниво $k$ в дървото.
- **Симeтрични дървета**: Дърво, чието ляво поддърво е огледален образ на дясното.

## Обобщение

| Концепция           | Описание                                                         |
| ------------------- | ---------------------------------------------------------------- |
| **AST**             | Дърво, представящо структурата на математически/програмен израз |
| **Environment**     | Списък от двойки `(променлива, стойност)` за оценка на изрази   |
| **BST свойство**    | Ляво < Корен < Дясно за всеки възел                              |
| **Листо (Leaf)**    | Възел без наследници                                             |
| **Ниво (Level)**    | Разстояние от корена (коренът е на ниво 0)                       |
| **Братя (Siblings)**| Възли с общ родител                                              |
| **Pruning**         | Премахване на всичко след дадено ниво `k`                        |
| **Ротация**         | Структурна промяна, запазваща наредбата                          |

| Операция        | Сложност (балансирано BST) | Сложност (общо) |
| --------------- | -------------------------- | --------------- |
| Търсене/Search  | O(log n)                   | O(n)            |
| Вмъкване/Insert | O(log n)                   | O(n)            |
| Изтриване/Delete| O(log n)                   | O(n)            |
| `isBST`         | O(n)                       | O(n)            |
| `getLeaves`     | O(n)                       | O(n)            |
| `allPaths`      | O(n)                       | O(n)            |
